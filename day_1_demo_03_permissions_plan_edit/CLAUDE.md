# CLAUDE.md — day_1_demo_03_permissions_plan_edit

## What this project is

A simple planning workspace with two versions of the same document:

- `docs/official/onboarding.md` — the **approved** plan. **Read-only.** Items 1–3 have been signed off. Do not edit this file directly.
- `docs/draft/onboarding.md` — the **working copy**. Contains the approved items plus two proposed additions (items 4–5) and a Pending Decisions section awaiting sign-off.

## Permission policy

Write and Edit operations on `docs/official/**` are **denied** by `.claude/settings.json`. The approved plan is locked — changes go through the draft first, then a human promotes them to `official/` after review.

If asked to update the plan, always target `docs/draft/onboarding.md`.

## How to apply a change

1. Edit `docs/draft/onboarding.md`.
2. Note what changed in a short summary.
3. A human reviewer promotes the draft to `official/` after approval.

## Conventions

- Markdown only — no HTML.
- Tables use GitHub-flavoured Markdown pipe syntax.
- Dates in `YYYY-MM-DD` format.
